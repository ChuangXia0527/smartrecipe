## 摘要（中英文）
- 研究背景：饮食场景中“选什么吃”决策成本高。
- 研究目标：构建支持图像识别、语音输入、用户偏好的食谱推荐系统。
- 方法概述：
  - 图像识别：TFLite 食材分类；
  - 语音理解：规则词典 + 否定识别；
  - 推荐：基于偏好、收藏、行为、烹饪时长与热量的加权排序。
- 结果：给出系统功能完成度与实验指标（后续补实验数据）。
- 结论：验证多模态输入提升推荐可用性与交互体验。

## 4. 答辩最容易被问的问题（附可用回答思路）

1. **你的“智能”体现在哪里？**  
   回答：多模态输入（图像+语音+偏好）与个性化加权推荐，而非单一关键词检索。

2. **为什么不用深度学习做端到端推荐？**  
   回答：毕业设计周期与数据规模受限，选用可解释、可控、可迭代的规则+行为加权方案；后续可升级到学习排序模型。

3. **你如何验证系统有效性？**  
   回答：通过功能测试 + 对照实验 + 用户满意度问卷三类证据。

4. **系统缺点是什么？**  
   回答：数据规模较小、语义解析依赖词典、模型泛化能力有限；已给出改进路线。
